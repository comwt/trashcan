Release Notes


The following changes were made for Trash Can version 2.0:

Installation Script:

  • Changed alot of things here:
    • Made installation smoother with better error handling.
    • Restore() function performs a full restore now if installation fails.
    • Instead of defaulting to purge trash older than 7 days the user is prompted for their preference right away.
    • Added the configurization of the new feature that handles Trash Can Capacity.
    • Version checking enabled.
    • trash.conf defaults are set during installation now.
    • File permissions via chmod are set for the trash directory and contents.
    • The profile choice has been revised completely, even offering the user to specify one other than those listed.
      Includes verification sequence.
    • More possible binary directory paths have been added for detection (for root install).

Executable:

  • Added ability to let USER change the storage capacity of their Trash Can.
  • A notification (soft warning) is displayed when you have reached 75% trash can capacity.
  • A notification (hard warning) is displayed when you have reached 100% trash can capacity and suggestions are given in order to clean out the can. You can no longer remove files through Trash Can until action is taken.
  • Changed a few program options to allow trash.sh to run on more platforms (sorry for any earlier inconvenience).

    Many thanks for the advice of one programmer in particular for pointing out Linux's need of the -P option with tar (not needed on SCO). This was, no doubt, causing many grief where on certain linux systems the purpose of TrashCan was thwarted by not having this option. Also for advising '^.*' instead of '^*' in trash.sh (same problem). Thanks Tim.

Un-Install Script:

  • Smoothed out flow a little.
  • You must first re-install Trash Can 2.0 over your previously installed version for it to work.
  • Re-Installing will not delete your old trash since the install script recognizes its existance.
